## V. VISUAL DESIGN DIRECTION

### Color Palette
- **Primary Surface**: Very dark neutral (#0a0a0a to #1a1a1a) — clean, premium
- **Secondary Surface**: Slightly lighter dark (#232323 to #2d2d2d) — for contrast and sections
- **Accent Primary**: Electric blue-violet (#4d63ff or similar) — restrained, high-contrast
- **Accent Secondary**: Subtle cyan/white (#e0f2ff) — for interactive states
- **Text Primary**: Near-white (#f5f5f5) — maximum readability on dark
- **Text Secondary**: Medium gray (#a0a0a0) — for supporting copy
- **Accent Emphasis**: White (#ffffff) — rare, high-impact moments only

### Typography System
- **Display Font**: Use something with personality but premium (e.g., Inter, Recursive, Satoshi, or similar geometric sans with character)
  - h1: 56px / 1.2 line-height / bold
  - h2: 40px / 1.25 / semi-bold
  - h3: 28px / 1.3 / semi-bold
- **Body Font**: Clean, highly readable sans (e.g., Inter, System UI stack)
  - body: 16px / 1.6 line-height
  - captions: 14px / 1.5
- **Line Length**: Max 65 characters for readability; break into columns on desktop if needed
- **Emphasis**: Use weight variation, size, and whitespace; avoid italic or all-caps

### Visual Language
- **Spacing**: Premium, generous white space; breathing room between sections
- **Borders**: Subtle 1px dividers (#333 or #444) only where necessary
- **Shadows**: Minimal; use only for layering (soft, directional, not heavy)
- **Motion**: Smooth, purposeful; fade-ins on scroll, gentle hover states, no distracting animations
- **Imagery**: 
  - Project screenshots/product mockups (real, not generic)
  - Optional: subtle abstract visuals or code snippets for technical sections
  - No stock photos
  - No gradients (keep surfaces flat and clean)
  - No glow effects unless absolutely minimal and purposeful

### Component Guidance
- **Buttons**: Bordered (outline style) on dark, filled on CTA; hover state = lighter border + subtle background
- **Cards**: Flat, with subtle 1px border; on hover: slight background lift, border brightens
- **Links**: Underline on hover; accent color by default
- **Forms**: Clean, minimal; accent color on focus; clear error states
- **Navigation**: Clean horizontal layout; active state indicated by underline or weight change
- **Footer**: Mirrors header style; includes brand mark, nav, social, copyright

### Responsive Approach
- **Mobile-first**: Design for mobile, enhance for desktop
- **Tablet**: Optimize readability; two-column layouts where sensible
- **Desktop**: Premium spacing, larger typography, full visual expression
- **No hamburger menus**: Use horizontal nav on mobile if possible; stack sections below on very small screens
